## 2. 지도 개선

### 핀 겹침 문제 해결
- **문제**: T1(홍대)과 삼소회동이 0.7px, 크래프톤·엔씨소프트가 2.1px 거리로 완전히 겹침
- **원인**: 두 이벤트가 실제로 같은 동네(홍대, 판교)에서 일어나서 지도 픽셀이 동일
- **해결**: 핀 displacement 알고리즘 구현 (`buildMap()` 내 `dp[]` 배열)
  - 두 핀이 `반지름A + 반지름B + 8px` 이내면 서로 반대 방향으로 밀어냄
  - 5회 반복으로 연쇄 밀림(홍대 → 판교 체인)까지 안정 수렴
  - 결과: T1↔삼소(39px), 크래프톤↔엔씨(40px)로 분리

### 시각적 위계 도입
- **완료된 일정**: 큰 번호 핀 → 작은 `✓` 점만 표시 (라벨 제거)
- **진행중**: 큰 글로우 핀 + 펄스 애니메이션 + 충돌 회피 라벨
- **예정**: 중간 크기 핀 + 4방향 라벨 배치 알고리즘
  - 상·하·좌·우 4방향 중 다른 핀과 덜 겹치는 방향에 라벨 자동 배치

## 3. 주가 섹션 개선

### 외국인·기관 순매수 칼럼 추가
- **데이터 출처**: `finance.naver.com/item/frgn.naver?code={종목코드}` HTML 파싱
- **파싱 방법**: EUC-KR 디코딩 → `<tr>` 날짜행에서 정규식 추출
  - 행 구조: `날짜|종가|방향|등락률|거래량|외국인순매수|기관순매수|보유주수|보유비율`
- **표시 단위**: 주 → 만주 변환 (예: -18,198주 → `-1.8만`)
- 관련 함수: `agents/stock-agent.js`의 `fetchNaverInvestor()`, `fmtInv()`, `fmtVol()`
- ⚠️ **리스크**: HTML 구조 변경 시 파싱 즉시 불가, Naver ToS 이슈 있음 → KRX API 전환 검토 필요

### 기준값 단순화
- **문제**: 방한확정比(5/27)와 뉴스比(6/1) 두 숫자가 같은 크기로 표시돼 혼란
- **변경**: 방한확정比를 Primary(크게), 뉴스比를 Secondary(작게)로 위계 분리
- **기능**: 오른쪽 작은 숫자(⇄) 클릭 시 Primary/Secondary 자리 교체
- 상태 저장: `window._basePrims` 객체에 종목별 현재 설정 유지 → 주가 갱신 후에도 사용자 선택 보존
- 관련 함수: `swapBase(code)`, `renderDual(baseEl, prim)`

## 4. 뉴스·UX 개선

### 종목/섹터 클릭 → 뉴스 필터
- 종목 카드 클릭 → 해당 종목 관련 뉴스만 필터링 후 뉴스 섹션으로 스크롤
- 섹터 헤더(반도체·HBM, 로봇·피지컬AI 등) 클릭 → 해당 섹터 전체 종목 관련 뉴스 필터링
- 히트맵 셀도 클릭 가능하게 연결
- 필터 상태 표시: `📌 LG전자 (066570) 관련 뉴스 N건 | × 필터 해제`
- 관련 함수: `filterNewsByStock(code, name)`, `filterNewsByTheme(themeIdx)`, `clearNewsFilter()`

### 오늘의 요약 브리핑 모달
- nav에 "📋 오늘 요약" 버튼 추가
- 클릭 시 모달 오픈: 현재 이벤트 → 일정 체크리스트 → TOP 5 종목 수익률 → AI 인사이트 요약
- `?brief=1` URL 파라미터로 직접 접근 가능 → 카카오/슬랙 공유 링크로 사용
- 🖨 인쇄 버튼: `@media print` CSS로 모달만 흰 배경 1페이지로 출력 (PB용)
- 🔗 공유 버튼: `?brief=1` URL 클립보드 복사
- 관련 함수: `openBrief()`, `closeBrief()`, `shareBrief()`, `buildBrief()`

### hero 이미지 겹침 해결
- **원인**: `position:absolute;bottom:0`으로 배치된 젠슨황 사진이 hero wrap보다 키가 클 때 위로 넘쳐 텍스트와 겹침
- **해결**: `position:absolute` 완전 제거 → flexbox 형제 관계로 재구성
  - `.hero .wrap { display:flex; align-items:flex-end; }` + `.hero-text { flex:1 }` + `.hero-img { align-self:flex-end }`
  - 이미지가 flex 형제로 들어가 overflow 자체가 불가능한 구조

### 시간 표시 버그 수정
- **문제**: 주가 카드의 업데이트 시각이 live API 응답 전까지 `--:--` 표시
- **해결**: `loadEvents()` 완료 시 `events.json`의 `updatedAt` 값으로 `STOCK_UPDATED_AT` 초기화
- 수집 현황 카드의 날짜도 함께 갱신 (기존: `06.03` 하드코딩)

## 5. 에이전트 시스템 구축

### 파일 구조
```
agents/
├── orchestrator.js   ← 전체 파이프라인 제어 (LLM 없음)
├── news-agent.js     ← RSS 수집 + Claude Haiku 뉴스 구조화
├── stock-agent.js    ← Naver API 주가 + 외국인/기관 수집
├── map-agent.js      ← events.json → 지도 데이터 변환
├── insight-agent.js  ← Claude Sonnet Pre/Post 인사이트 생성
└── info-agent.js     ← meta.json 이력 관리
scripts/
└── batch-runner.js   ← GitHub Actions 진입점
```

### LLM 사용 구조
| 에이전트 | 모델 | 역할 |
|---|---|---|
| News Agent | claude-haiku-4-5-20251001 | 기사 → 이벤트 카드 구조화, 완료 여부 판단 |
| Insight Agent | claude-sonnet-4-6 | Pre/Post 인사이트 JSON 생성 |
| 나머지 3개 | LLM 없음 | 수치 계산·변환만 수행 |

- ANTHROPIC_API_KEY 없으면 LLM 단계 자동 스킵, 나머지 정상 동작

### GitHub Actions 자동 배치
- `.github/workflows/news-update.yml`: `cron '*/30 * * * *'` (30분 주기)
- 실행 결과를 `events.json`에 반영 후 자동 커밋·푸시 → Vercel 자동 재배포
